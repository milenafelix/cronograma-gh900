# Alimentar a IA

**Pergunta 1: Qual das seguintes opções é uma prática recomendada ao usar o GitHub Discussions para um projeto de código aberto?**

- **Resposta:** Incentive os colaboradores a usar a seção de Discussões para brainstorming e colaboração antes de abrir novas questões.

**Pergunta 2: Você está revisando as alterações de código que um colega fez em uma ramificação de recurso. Você percebe que as alterações introduzem uma potencial vulnerabilidade de segurança. Qual é a abordagem recomendada para lidar com esse problema durante o processo de revisão de código?**

- **Resposta:** Solicite alterações no pull request e forneça um feedback detalhado explicando a preocupação com a segurança e como resolvê-la.

**Pergunta 3: Você preparou algumas novas alterações em seu repositório Git e deseja confirmá-las com uma mensagem. Qual comando você usaria para confirmar as alterações preparadas com a mensagem de commit "Corrigido bug na função principal"?**

- **Resposta:** git commit -m "Corrigido bug na função principal"

**Pergunta 4: Você criou uma nova branch de funcionalidade no seu repositório GitHub e deseja mesclá-la à branch principal. Após revisar as alterações, você decide iniciar um pull request. Qual das seguintes etapas você deve executar em seguida?**

- **Resposta:** Atribua revisores à solicitação de pull request.

**Pergunta 5: Você deseja criar uma cópia local de um repositório Git hospedado em um servidor remoto. Qual comando Git você deve usar?**

- **Resposta:** git clone https://docs.github.com/pt/repositories

**Pergunta 6: Você é o administrador de um tenant do Microsoft Entra ID e precisa configurar uma organização do GitHub para a equipe de desenvolvimento da sua empresa. Qual das seguintes etapas é necessária para habilitar a integração do Microsoft Entra ID com a sua organização do GitHub?**

- **Resposta:** Instale o aplicativo GitHub Entra ID Connector a partir do GitHub Marketplace.

**Pergunta 7: Você tem um repositório Git local com várias branches. Você deseja combinar as alterações da branch 'feature' na branch 'main'. Qual comando Git você deve usar para realizar essa tarefa?**

- **Resposta:** git merge feature

**Pergunta 8: Qual das seguintes opções descreve melhor o recurso de Issues do GitHub?**

- **Resposta:** Uma forma de rastrear bugs e ideias de melhoria para um repositório, com etiquetas, responsáveis e marcos.

**Pergunta 9: Você deseja colaborar em um projeto hospedado no GitHub criando sua própria cópia do repositório para fazer alterações livremente sem afetar o repositório original. Qual ação você deve tomar?**

- **Resposta:** Faça um fork do repositório para sua própria conta do GitHub.

**Pergunta 10: Você deseja contribuir para um projeto de código aberto hospedado no GitHub. Qual das seguintes ações você deve tomar para criar sua própria cópia do repositório que possa ser modificada sem afetar o projeto original?**

- **Resposta:** Faça um fork do repositório para sua conta do GitHub.

**Pergunta 11: Você deseja propor uma alteração em um repositório do GitHub no qual não possui permissão de escrita. Para fazer isso, você deve:**

- **Resposta:** Crie um novo fork do repositório e envie uma solicitação de pull request a partir desse fork.

**Pergunta 12: Um desenvolvedor deseja garantir que vulnerabilidades de segurança em dependências sejam identificadas e resolvidas automaticamente, criando solicitações de pull para atualizar as dependências para versões seguras. Qual recurso de segurança do GitHub deve ser habilitado?**

- **Resposta:** GitHub Dependabot.

**Pergunta 13: Você fez algumas alterações em um arquivo no seu repositório Git local. Para adicionar as alterações à área de preparação (staging area) e criar um novo commit, qual combinação de comandos Git você deve usar?**

- **Resposta:** git add . && git commit -m "mensagem de commit"

**Pergunta 14: Ao adicionar um novo campo a um quadro de projeto do GitHub, qual das seguintes opções permite personalizar o campo com opções predefinidas?**

- **Resposta:** Lista de seleção única.

**Pergunta 15: Você deseja analisar o trabalho realizado em uma grande atualização. Qual das seguintes combinações de recursos do GitHub você deve usar para revisar os problemas e as solicitações de pull associadas a essa atualização?**

- **Resposta:** Rótulos e marcos.

**Pergunta 16: Você deseja configurar regras de proteção para a branch de produção. Quais das seguintes configurações você habilitaria como parte dessas regras?**

- **Resposta:** Todas as opções são verdadeiras (Exigir pull request, exigir revisão de código e não permitir exclusão da branch).

**Pergunta 17: Você precisa acompanhar o progresso de um projeto de desenvolvimento de software usando o GitHub Projects. Qual dos seguintes recursos você pode usar para visualizar o fluxo de trabalho do projeto?**

- **Resposta:** Quadros de projetos do GitHub.

**Pergunta 18: Você precisa compartilhar um repositório privado com membros da equipe que possuem diferentes níveis de acesso. Qual é a maneira mais segura de gerenciar as permissões?**

- **Resposta:** Crie o repositório, crie uma equipe e atribua à equipe as permissões apropriadas para acessar o repositório.

**Pergunta 19: Qual opção descreve com precisão a finalidade da aba "Pull requests" na interface do repositório GitHub?**

- **Resposta:** Para iniciar um processo de mesclagem de alterações de código de outra ramificação ou fork na ramificação atual.

**Pergunta 20: Você deseja garantir que todos os colaboradores tenham permissões iguais para gerenciar um repositório público em um projeto open source. Qual etapa deve seguir?**

- **Resposta:** Crie um novo repositório público e adicione colaboradores com o nível de acesso "Administrador".

**Pergunta 21: Qual das seguintes opções descreve melhor a finalidade do recurso "Problemas" (Issues) na interface do GitHub?**

- **Resposta:** Utilizado para rastrear e gerenciar tarefas, solicitações de recursos e bugs relacionados a um repositório.

**Pergunta 22: Um novo membro precisa de acesso a vários repositórios de um projeto. Qual ação você deve tomar para conceder o acesso apropriado?**

- **Resposta:** Crie uma nova equipe para o projeto, adicione o membro a essa equipe e configure as permissões de acesso ao repositório da equipe.

**Pergunta 23: Qual das seguintes opções é a maneira mais apropriada de interagir com os mantenedores e a comunidade de um projeto de código aberto?**

- **Resposta:** Abra uma issue no repositório para discutir suas ideias e obter feedback antes de fazer alterações.

**Pergunta 24: Qual comando Git você deve usar para atualizar seu repositório local com as alterações mais recentes do repositório remoto (buscando e mesclando)?**

- **Resposta:** git pull origin master

**Pergunta 25: Em uma equipe que usa o GitHub, qual recurso você deve usar para facilitar a discussão e a visibilidade das alterações de código propostas?**

- **Resposta:** Solicitações de Pull Requests.

**Pergunta 26: Você precisa criar um novo repositório no GitHub para armazenar o código do projeto, gerenciar o acesso e rastrear as alterações. Qual das seguintes ações você deve realizar primeiro?**

- **Resposta:** Crie um novo repositório em sua conta do GitHub existente.

**Pergunta 27: Você criou uma nova branch e confirmou alterações. Qual é o próximo passo para propor essas alterações para revisão e integração na branch principal?**

- **Resposta:** Crie um novo pull request da branch de funcionalidade para a branch principal.

**Pergunta 28: Alguns membros da equipe estão enfrentando problemas ao enviar alterações para branches protegidas. Qual configuração resolve isso com segurança?**

- **Resposta:** Adicione o grupo de IDs do Microsoft Entra que contém os membros da equipe à lista de colaboradores autorizados para o repositório.

**Pergunta 29: Você deseja automatizar a criação de um novo projeto sempre que uma nova issue for criada com um rótulo específico. Qual recurso deve usar?**

- **Resposta:** GitHub Actions.

**Pergunta 30: Você precisa conceder acesso somente leitura a uma equipe de consultores externos. Qual opção permite isso garantindo a segurança?**

- **Resposta:** Crie uma nova função de repositório com permissões de leitura e atribua-a às contas Microsoft Entra ID dos consultores.

**Pergunta 31: Qual das seguintes opções NÃO é um princípio fundamental do DevOps?**

- **Resposta:** Equipes de desenvolvimento e operações isoladas.

**Pergunta 32: Qual das seguintes afirmações sobre repositórios Git é verdadeira?**

- **Resposta:** Um repositório Git é um diretório local no seu computador que armazena arquivos e registra as alterações feitas neles.

**Pergunta 33: Qual das seguintes opções NÃO é um gatilho de fluxo de trabalho válido do GitHub Actions?**

- **Resposta:** Ao alterar um arquivo em um diretório específico. *(Nota: O gatilho correto seria baseado em eventos como push/pull_request filtrados por paths, mas o evento de "alteração de arquivo" isolado não é um gatilho 'top-level' como os outros).*

**Pergunta 34: Qual das seguintes opções descreve melhor um repositório Git?**

- **Resposta:** Um local central onde todos os arquivos e códigos de um projeto são armazenados e gerenciados.

**Pergunta 35: Você fez várias alterações e deseja prepará-las para o próximo commit. Qual comando usaria?**

- **Resposta:** git add .

**Pergunta 36: Qual das seguintes opções descreve melhor o fluxo de trabalho básico do Git?**

- **Resposta:** Criar branch de funcionalidade, confirmar as alterações na branch, mesclar na branch principal e enviar para o repositório remoto.

**Pergunta 37: Qual é o próximo passo para permitir que outras pessoas revisem e forneçam feedback sobre suas alterações antes de mesclá-las à branch principal?**

- **Resposta:** Envie sua branch para o repositório remoto e crie uma solicitação de pull request.

**Pergunta 38: Qual das seguintes afirmações descreve com precisão a diferença entre um repositório local e um repositório remoto?**

- **Resposta:** Um repositório local é armazenado no seu computador, enquanto um repositório remoto é armazenado em um servidor e permite a colaboração.

**Pergunta 39: Qual das seguintes afirmações descreve melhor por que o Git é usado?**

- **Resposta:** O Git permite que vários desenvolvedores trabalhem simultaneamente na mesma base de código, rastreando alterações e mesclando código de diferentes branches.

**Pergunta 40: Você deseja contribuir para um projeto de código aberto. Qual seria a melhor maneira de começar tecnicamente?**

- **Resposta:** Faça um fork do repositório, implemente suas alterações localmente e envie uma solicitação de pull request.
